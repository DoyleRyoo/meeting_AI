# github 사용 시 유의사항
### 모든 규칙은 호의를 통해 수정될 수 있음을 알립니다.
## github 사용 규칙

### 1. main에 절대 push하지 않는다.
### 2. 다른 팀원 branch를 사용 시 자신의 새 branch를 만들어서 사용한다. 
### 3. full request할 때 과반수 동의하에 진행한다.
- 코드 오류 발견하기 위해
### 4. 버전이 한 개 완성될 때마다 main에 full request한다.
- ver. 1.0.0이 ver. 1.0.1처럼 숫자가 변경되는 것
### 5. 버전 업데이트 시 README.md에 버전과 무엇이 업데이트 되었는지 요약하여 적는다.
- ver. 1.2.3 : 녹음이 텍스트로 변환되지 않는 오류 수정

## 버전 관리

소프트웨어 버전은 **`주버전.부버전.패치버전`** 형태로 표기한다.

현 프로젝트는 ver. 1.0.0부터 사용하며 ver. 2.0.0으로 올라지 않는다.

main으로 pull request가 되어도 추후에 버그가 발견될 수 있기 때문에 패치 버전도 사용한다.

예를 들어 `ver. 2.15.3`이라는 버전이 있다면 다음과 같은 의미를 가진다.

| 버전 | 의미 | 변경 내용 |
| --- | --- | --- |
| **2** | 주 버전 (Major) | 기존 버전과 호환되지 않는 큰 변화 |
| **15** | 부 버전 (Minor) | 새로운 기능 추가, 기존 기능과의 호환성 유지 |
| **3** | 패치 버전 (Patch) | 버그 수정, 성능 개선, 보안 업데이트 |

### 예시

#### `1.0.0`
- 1번 목표까지 완성되었을 시 첫 정식 출시 버전으로 간주

#### `1.1.0`
- 기존 기능과의 호환성이 유지되는 새로운 기능이 추가되었을 경우
- 2번과 같은 기능이 추가 되었을 시 중간 버전을 올림

#### `1.1.1`
- 버그 수정만 진행

#### `2.0.0`
- 현 프로젝트에서는 올라가지 않을 버전
